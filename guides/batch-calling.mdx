## Overview

TalkifAI's Batch Calling system lets you run large-scale outbound campaigns. Built on **Redis + ARQ**, it supports 100k+ calls/day with automatic retry, timezone-aware scheduling, and two-level concurrency control.

---

## How It Works

```
Batch Job Created
      │
      ▼
Jobs stored in Redis queue
      │
      ▼
ARQ Worker picks up jobs
      │
      ├── Check Org concurrency limit (default: 5 concurrent)
      ├── Check Batch concurrency limit (default: 3 concurrent)
      │
      ▼
   Call initiated
      │
      ├── Success → Mark complete
      └── Failure → Retry (up to 3 times)
```

---

## Concurrency Control

Batch calling uses **two-level concurrency control** (similar to Retell AI):

```
Level 1: Organization Limit
         └─ Max concurrent calls across ALL batches in your org (default: 5)

Level 2: Batch Limit
         └─ Max concurrent calls within a single batch (default: 3)
```

**Example:**
- Your org limit: `5`
- Batch A limit: `3`
- Batch B limit: `4`

At any moment, your org can run max **5** concurrent calls total — not 7.

## Create a Batch

```bash
POST /v1/batch-calls
```

```json
{
  "name": "January Renewal Campaign",
  "agentId": "agent_abc123",
  "concurrentCallLimit": 3,
  "scheduledTime": "2024-01-20T09:00:00",
  "timezone": "America/New_York",
  "contacts": [
    {
      "phoneNumber": "+12025550101",
      "variables": {
        "customerName": "John Smith",
        "renewalDate": "Feb 1"
      }
    },
    {
      "phoneNumber": "+12025550102",
      "variables": {
        "customerName": "Jane Doe",
        "renewalDate": "Feb 5"
      }
    }
  ]
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Batch identifier |
| `agentId` | Yes | Agent that handles calls |
| `contacts` | Yes | List of phone numbers + variables |
| `concurrentCallLimit` | No | Max parallel calls (default: 3) |
| `scheduledTime` | No | When to start (omit for immediate) |
| `timezone` | No | IANA timezone (default: UTC) |

**Response:**
```json
{
  "success": true,
  "data": {
    "batchId": "batch_xyz789",
    "name": "January Renewal Campaign",
    "status": "scheduled",
    "totalCalls": 2,
    "scheduledTime": "2024-01-20T14:00:00Z",
    "createdAt": "2024-01-15T10:30:00Z"
  }
}
```

## Per-Contact Variables

Inject dynamic data into your agent's system prompt using contact variables:

In your **Agent System Prompt**:
```
You are calling {{customerName}} about their renewal on {{renewalDate}}.
Greet them by name and confirm the renewal date.
```

Variables from the batch contact are automatically substituted at call time.

## Managing Batches

### Get Batch Status

```bash
GET /v1/batch-calls/{batchId}
```

```json
{
  "success": true,
  "data": {
    "batchId": "batch_xyz789",
    "status": "running",
    "progress": {
      "total": 100,
      "completed": 42,
      "failed": 3,
      "pending": 55,
      "running": 3
    },
    "startedAt": "2024-01-20T14:00:05Z"
  }
}
```

### Pause a Batch

```bash
POST /v1/batch-calls/{batchId}/pause
```

### Resume a Batch

```bash
POST /v1/batch-calls/{batchId}/resume
```

### Cancel a Batch

```bash
POST /v1/batch-calls/{batchId}/cancel
```

### Start Immediately (Override Schedule)

```bash
POST /v1/batch-calls/{batchId}/start-now
```

## Retry Behavior

Failed calls are automatically retried:
- **Attempt 1**: Immediate
- **Attempt 2**: After 5 minutes
- **Attempt 3**: After 30 minutes
- **After 3 failures**: Call marked as `permanently_failed`

Common failure reasons: no answer, busy, network error.

## Best Practices

<AccordionGroup>
  <Accordion title="Start with a small test batch">
    Before running 10,000 calls, test with 10-20 contacts to verify your agent handles the conversations correctly.
  </Accordion>
  <Accordion title="Respect calling hours">
    Use `scheduledTime` + `timezone` to ensure calls are made during business hours in the recipient's timezone. Most jurisdictions prohibit calls before 8 AM or after 9 PM local time.
  </Accordion>
  <Accordion title="Set appropriate concurrency">
    Higher concurrency = faster campaign completion but more load on your systems. Start with 3 and increase based on your infrastructure.
  </Accordion>
  <Accordion title="Monitor failed calls">
    Check your batch status regularly. A high failure rate (>20%) may indicate a carrier issue or incorrect phone numbers.
  </Accordion>
</AccordionGroup>
